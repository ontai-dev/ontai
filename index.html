The Problem

Documentation rots

The moment you publish it, it drifts. Every wiki page, every architecture diagram, every runbook is already describing a system that has moved on.

Operators are islands

Thousands of Kubernetes operators exist. None of them share a semantic contract surface. No common language. No lineage. No way to compose governance across domain boundaries.

AI cannot help an unstructured environment

AI in production operations requires semantic structure, causal memory, and an enforced approval boundary. Most platforms have none of these. ONT builds all three.

What ONT Does

ONT is a living documentation system first. Infrastructure governance is the consequence, not the purpose.

When every organizational decision is a CRD, every policy a versioned resource, every contract a reconciled object, the cluster holds the organizational truth with an API that any human, operator, or AI agent can query at any moment.

  • Queryable: ask the cluster what is true right now
  • Diffable: ask what changed and when
  • Auditable: ask who declared it and why
  • Living: reflects runtime reality, not someone's last memory

The Seam Operator Family

Guardian

Trust root and security substrate. Owns RBAC governance, PermissionSnapshot computation, and the audit sink.

Platform

Cluster lifecycle authority. Imports and bootstraps Talos Linux clusters. Auto-onboards tenant clusters.

Wrapper

Pack delivery engine. OCI artifact lifecycle from ClusterPack signing through PackInstance creation.

Conductor

Execution intelligence. Two binaries: Compiler (never deployed) and Conductor agent (distroless, everywhere).

seam-core

Lineage and semantic DNS controller. Maintains the InfrastructureLineageIndex and the DSNS zone.

domain-core

Domain primitive declarations. The nine abstract primitives every ONT domain inherits from.
